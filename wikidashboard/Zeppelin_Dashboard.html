Etape 1 : Exécution de la requête[edit]

Remplissage du formulaire[edit]

Pour générer un bilan il suffit de remplir le formulaire se trouvant en haut de la page :



Les champs à remplir sont les suivants :

  • idm : ID du Mailing
  • id_tag : ID du Tag
  • délai : Le délai souhaité en nombre de jours (Ex : 30)
  • ddebut : Date de début de la recherche en format YYYY-MM-DD (Ex : 2018-01-01)
  • dfin : Date de fin de la recherche en format YYYY-MM-DD (Ex : 2018-02-02)
  • tache : Le type des éléments recherchés. Veuillez choisir entre conversions et commandes

IMPORTANT : Veuillez laisser les guillemets entre lesquels se trouvent les valeurs. (Pour l’idm 17652, la valeur fournie doit être ‘17652’ et pas 17652).

Exécution du paragraphe[edit]

Après avoir rempli les différents champs, il suffit de lancer le paragraphe qui contient le formulaire :



Le temps d'exécution varie largement selon plusieurs critères :

  • Si c’est une recherche qui a été lancée avant par vous même ou bien par un autre utilisateur : résultat immédiat
  • Si c’est une nouvelle recherche (l’ensemble des 6 champs ne correspond à aucune ancienne requête), le temps d’exécution sera entre 7 et 16 minutes approximativement selon le volume de données, la charge sur les différents serveurs, la date de début et la date de fin de votre recherche, et d’autres critères.


Lorsque l'exécution de la requête se termine avec succès, vous aurez le message suivant en dessous du formulaire :


Gestion des erreurs[edit]

Si un message d’erreur s’affiche, la solution dépendra du type de message :

  • Veuillez tout d’abord vérifier les valeurs insérées dans les différents champs du formulaire, et que chaque valeur est écrite entre des guillemets simples (‘17652’ et pas 17652 / ‘17652 / 17652’ / "17652")
  • Si l’erreur n’est pas due aux valeurs du formulaire, veuillez redémarrer l’interpréteur msdb_in_1_sql1. Pour le faire, veuillez suivre les deux étapes suivantes :
  1. Veuillez cliquer sur l’icône en forme d'engrenage en haut de page
  2. Veuillez cliquer sur l’icône “Restart” devant msdb_in_1_sql1 pour le redémarrer
  • Si le problème persiste, veuillez contacter Mahdi en lui envoyant le message d’erreur.

Téléchargement des données[edit]

Pour chaque bilan, les données peuvent être téléchargée en format CSV ou TSV. Pour faire celà, il suffit de cliquer sur l'icône de téléchargement :



Choix du mode de l'affichage[edit]

Zeppelin vous propose trois modes d'affichage pour votre bilan :

  • Mode par défaut
  • Mode simple (élimination de la séparation entre les différents rapports)
  • Mode rapport (mode statique qui affiche le dashboard sous format de rapport en cachant les différents paramètres de configuration)

Pour choisir entre ces modes, il suffit de cliquer sur le mode souhaité grâce à la liste déroulante en haut de la page :
